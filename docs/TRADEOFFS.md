# Raven Tradeoffs

Raven is intentionally asymmetric: local work over cloud convenience, reliable playback over silence, freshness over perfect continuity, and subscription compatibility over maximum API capability. This document makes those costs explicit. For the chronological context, see [`HISTORY.md`](HISTORY.md); for binding decisions, see the [`adr/`](adr/README.md) index.

| Tradeoff | Side A | Side B | Raven's choice and why |
| --- | --- | --- | --- |
| Voice quality vs. cost | ElevenLabs was more natural and suitable for companionship, but estimated at roughly $288–300/month for about 60 hours of monthly use, with a 100-conversation/30-day cap on the evaluated path. | Kokoro-82M is prosodically flat but runs locally on the M4 Pro for approximately $0 marginal cost. | **Kokoro.** Raven needs competent narration, not a companion persona. Zero marginal cost, local privacy, and sustainable daily use outweighed more expressive prosody. See [ADR 0004](adr/0004-local-kokoro.md). |
| Fresh content vs. jarring cuts | Let the current reply finish cleanly, even when it has become stale and may run for minutes. | Interrupt immediately when a newer selected-session reply arrives, potentially in the middle of a word or sentence. | **Latest wins, designed but deferred.** Freshness matters more during an active driving workflow. A later sentence-boundary policy can make automatic cuts less jarring without allowing multi-minute backlog. See [ADR 0010](adr/0010-latest-wins-interrupt.md), [ADR 0010](adr/0010-latest-wins-interrupt.md), and [`SCOPE_SENTENCE_CUT.md`](SCOPE_SENTENCE_CUT.md). |
| Time-to-first-word vs. double-speak | After a fixed timeout, race slow Kokoro with fast macOS `say`; speech starts sooner when synthesis is stuck, but a merely slow render can be spoken twice. | Wait while a live Kokoro daemon finishes, accepting up to about 15 seconds of hiss for a roughly 2,500-character reply. | **Wait for a live daemon.** The old five-second fallback produced two voices for one reply. Fallback now requires evidence that `synthd` is dead. Sentence-streamed synthesis is the real latency fix. See [`SCOPE_STREAMING_SYNTHESIS.md`](SCOPE_STREAMING_SYNTHESIS.md). |
| Background survival vs. silence | Emit digital silence between replies for a cleaner cabin. | Emit a quiet, audible pink-noise floor that keeps the background player and car audio path active. | **Comfort noise.** Digital silence allowed suspension and car-amplifier sleep, clipping the first word. The hiss is intentional and load-bearing. See [ADR 0003](adr/0003-continuous-hls-comfort-noise.md). |
| Subscription vs. capability | Use the Agent SDK with an `ANTHROPIC_API_KEY`, separate credit billing, and token-level partial messages. | Stay on the Claude.ai subscription with Claude Code hooks, which provide complete messages at lifecycle boundaries but no token deltas. | **Subscription-safe hooks.** Remote Control already drives the same Claude Code session, and the hook fires for those turns. Raven accepts that token-level streaming is unavailable. See [ADR 0005](adr/0005-claude-code-hooks.md) and [ADR 0011](adr/0011-no-token-streaming.md). |
| Native maintenance vs. PWA simplicity | A PWA is easier to distribute and avoids code signing, but it cannot own reliable background microphone capture and gives Raven less direct control over player retry, audio sessions, Now Playing, and the car route. | A native SwiftUI app requires signing, installation, and iOS-specific maintenance. | **Native.** The original full-duplex concept required capabilities web content cannot declare; after the voice-out reframe, reliable background playback and recovery still justified `AVPlayer` and `AVAudioSession`. This is an iPhone player, not a fork of Claude Code—the archived Omnara wrapper remains a warning against coupling Raven to Claude's client release cadence. See [ADR 0002](adr/0002-native-ios-app.md) and [ADR 0008](adr/0008-cli-ios-build.md). |
| Full automation and open-mic convenience vs. safety | A fully automatic, open-mic hands-free workflow minimizes interaction while driving. | Permission gates and a constrained working copy reduce the blast radius but introduce friction and limit what can happen unattended. | **Flagged, not silently accepted.** Asif stated a preference for full-auto plus open-mic use, but that is the riskiest combination. A git worktree boundary and permission gating were recommended. This is a safety posture for the broader Claude workflow, not a claim that Raven itself opens the microphone—Raven remains voice-out only. See [ADR 0001](adr/0001-voice-out-only.md). |

## Secondary operational tradeoffs

These are smaller but still deliberate:

- **Whole answers over short waits.** `MAX_SPOKEN_CHARS=0` prevents arbitrary byte cuts, at the cost of long synthesis and listening time. Streaming synthesis addresses startup; optional summarization addresses duration. See [ADR 0012](adr/0012-uncapped-replies.md) and [`SCOPE_SUMMARIZATION.md`](SCOPE_SUMMARIZATION.md).
- **One owner of the car route over audio mixing.** A non-mixable `.playback` session makes Raven the Now Playing app and avoids drive-long ducking, but starting Spotify can take the session. See [ADR 0006](adr/0006-non-mixable-playback.md).
- **Private fixed topology over public reachability.** Tailscale eliminates a relay and application authentication layer, but both devices must remain on the tailnet and the current iPhone build hard-codes the Mac address. See [ADR 0007](adr/0007-tailscale-transport.md).
- **Compatibility over a clean rewrite.** The Go port kept file formats and behavior compatible with the Bash/Python path and used parity tests. That preserved rollback and reduced migration risk, but retained some byte-cap and local-filesystem conventions. See [ADR 0009](adr/0009-go-orchestration-python-synthesis.md).
- **HLS durability over conversational latency.** One-second segments and `AVPlayer` buffering produce roughly 3–5 seconds of downstream delay, but the continuous live stream survives background playback and car-route changes more reliably than per-reply file handoffs.

## What these choices rule out

Raven is not a general voice assistant, a Claude Code wrapper, a cloud voice service, or a token-streaming API client. It is a private, local-first return audio path for an existing Claude Code session. That narrowness is what made the system verifiable after v1 failed.
